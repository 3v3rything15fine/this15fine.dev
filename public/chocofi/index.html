Chocofi Wireless — keycap + layer reference

A self-contained cheat sheet for a Chocofi Wireless (nice!nano v2, ZMK firmware). Use it to place keycaps on the base layer, look up what each hold-layer does, and pair across laptop, desktop, iPhone, and iPad. Jump to: keycap placement, layers, thumb cluster, pairing, troubleshooting.

1. Keycap placement (BASE layer)

What each physical position types when you press nothing else. Left half on the left, right half on the right. Thumb cluster beneath each half. If your caps are blank, this tells you what they do. If they are legended, place them accordingly.

/ (bottom-right corner): tap /, hold for right Shift.

BT0-BT4 = profile select. BTCLR wipes the current profile's bond (do before re-pairing).

BT_P / BT_N = previous / next profile. USB / BLE = output selector.

RST = soft reset. BOOT = bootloader for reflash.

SPTLT = Cmd+Space (iOS Spotlight). ALT-T = Cmd+Tab (iOS app switcher).

3. Thumb cluster legend

Six thumb keys: three per hand, arranged outer / middle / inner. Inner thumbs sit under your thumb at rest. Middle thumbs reach slightly out. Outer thumbs need a deliberate stretch.

Left hand

Outer — SHFT
hold for Shift.
Middle — TAB
tap = Tab. hold = NAV layer.
Inner — SPACE
tap = Space. Rests under thumb.

Right hand

Inner — ENTER
tap = Enter. Rests under thumb.
Middle — BSPC
tap = Backspace. hold = SYM layer.
Outer — DEL
tap = Delete. hold = NUM layer.
TRI-LAYER: hold left-middle (NAV) + right-middle (SYM) at the same time to reach the FUN / Bluetooth layer. Release either to drop back.

4. Pairing cheatsheet

Profile allocation

ProfileDeviceFUN-layer key
BT0Framework Desktop — paired ✓left-pinky home row (A)
BT1Framework Laptop 13left-ring home row (S)
BT2iPadleft-middle home row (D)
BT3iPhoneleft-index home row (F)
BT4Spareleft-index-stretch (G)

ZMK holds one bond per profile. All five hosts can be powered on and the keyboard shows connected to several, but only the active profile receives keystrokes. Switching is deterministic.

First pairing — any device

  1. Hold NAV+SYM to reach FUN. Tap the target profile (BT0-BT4).
  2. If that profile was used before, tap BTCLR to wipe the existing bond. The keyboard starts advertising.
  3. On the host: open Bluetooth settings. Device appears as nice!nano (or whatever CONFIG_ZMK_KEYBOARD_NAME is set to).
  4. Tap to pair. No PIN prompt — ZMK uses HID-over-GATT (passwordless).
  5. Host shows Connected. Type in a text field to confirm.
iOS GOTCHA: iOS sends raw USB HID codes through whatever Hardware Keyboard layout you set. Go to Settings › General › Keyboard › Hardware Keyboard and force the layout to U.S.. Do not leave it on Automatic. Wrong setting makes @ print " and other surprise swaps.

Switching between devices

Hold NAV+SYM, tap the profile number. iOS and iPadOS cache the bond and usually reconnect in under two seconds. macOS and Linux behave similarly.

When pairing fails

  1. On the host: Forget This Device.
  2. On the keyboard: FUN layer, tap BTCLR to wipe the profile bond.
  3. Power-cycle both halves of the Chocofi (toggle the power switch on each side).
  4. Retry from step 1 of First Pairing.

Known iOS / iPadOS issues

  • Dead keys after sleep/wake. Stuck Cmd or Shift after host wakes up. Toggle Bluetooth off/on in Control Center, or briefly &bt BT_DISC N then reconnect.
  • Voice Control / Switch Control on. These hijack modifiers. Check Accessibility settings.
  • Globe key. ZMK's GLOBE approximates Apple Globe: cycles keyboards, reaches dictation and emoji. It does not replicate the full hardware Fn behavior of a Magic Keyboard.
  • iPad bonus: hold Cmd in any app for its shortcut cheat sheet. Enable Full Keyboard Access under Accessibility › Keyboards if it does not appear.

5. Troubleshooting quick hits

  • Soft reset. Double-tap the reset button on the nice!nano. First tap goes to bootloader, second tap cancels back to normal boot. A single tap resets.
  • Enter bootloader for reflash. Double-tap reset. The half mounts as a USB drive named NICENANO. Drop a .uf2 in; the half auto-reboots.
  • Bonds wedged after a flash. BTCLR on the affected profile, then Forget on the host, then re-pair. Order matters — the host must not hold a stale key.
  • All bonds stuck. Flash the settings_reset UF2 (from the ZMK module) to wipe the settings partition. Re-flash your normal firmware after. Every profile has to be re-paired.
  • Only one half types. Central is the left half by default. Check batteries or power switches. If the peripheral (right) is dead, the left half still types its own keys. If the central is dead, nothing types.
  • Both halves typing but wrong symbols on iOS. Hardware Keyboard layout is set wrong. See the iOS gotcha callout above.
  • BLE refuses to advertise. Confirm output is set to BLE on the FUN layer (not USB). Then BTCLR, then retry.
